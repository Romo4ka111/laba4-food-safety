# Тестирование

## Функциональное тестирование

| Сценарий | Ожидаемый результат |
|---|---|
| Регистрация нового пользователя | пользователь создаётся, пароль сохраняется в виде хеша |
| Вход с корректными данными | сервер возвращает JWT и данные пользователя |
| Вход с неверным паролем | ответ 401, событие записывается в журнал |
| Просмотр списка проверок | отображается таблица/карточки, работает пагинация |
| Поиск проверки | список фильтруется по названию, предприятию или продукции |
| Фильтрация по риску/статусу | отображаются только подходящие записи |
| Сортировка списка | записи отображаются в выбранном порядке по дате, названию, предприятию, продукции, риску или статусу |
| Выгрузка CSV | скачивается файл `inspections.csv`, событие фиксируется в журнале |
| Добавление проверки | создаётся новая запись, событие попадает в журнал |
| Редактирование проверки | запись обновляется, событие попадает в журнал |
| Удаление проверки | запись удаляется после подтверждения, событие попадает в журнал |
| Выход из системы | токен удаляется из localStorage, пользователь возвращается на вход |

## Тестирование безопасности

| Проверка | Ожидаемый результат |
|---|---|
| Запрос `/api/inspections` без токена | 401 Unauthorized, событие AUTH_REQUIRED фиксируется в журнале |
| Запрос `/api/users` или `/api/admin/users` обычным пользователем | 403 Forbidden |
| Запрос `/api/audit-log` не администратором | 403 Forbidden, событие ACCESS_DENIED фиксируется в журнале |
| Попытка SQL-инъекции в поиске | данные не повреждаются, запрос параметризован |
| Попытка XSS в текстовом поле | текст отображается как строка, не выполняется как HTML |
| Проверка пароля в БД | открытого пароля нет, хранится `password_hash` |
| Блокировка пользователя | заблокированный пользователь не может войти |

## Команды для ручной проверки API

```bash
curl -i http://localhost:5001/api/health
```

Вход:

```bash
curl -X POST http://localhost:5001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"login":"admin","password":"Admin12345"}'
```

Запрос без токена:

```bash
curl -i http://localhost:5001/api/inspections
```

Ожидается `401 Unauthorized`.

Проверка сортировки и выгрузки после входа:

```bash
curl -H "Authorization: Bearer <TOKEN>" \
  "http://localhost:5001/api/inspections?sortBy=name&sortOrder=asc"

curl -H "Authorization: Bearer <TOKEN>" \
  "http://localhost:5001/api/inspections/export.csv?riskLevel=Высокий"
```
